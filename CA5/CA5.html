import string
from nltk.tokenize import word_tokenize
from nltk.tag import pos_tag

stopwords = nltk.corpus.stopwords.words("english")
lemmatizer = nltk.stem.WordNetLemmatizer()
valid_words = set(nltk.corpus.words.words())

def nltk_pos_tagger(nltk_tag):
    if nltk_tag.startswith('J'):
        return 'a'
    elif nltk_tag.startswith('V'):
        return 'v'
    elif nltk_tag.startswith('N'):
        return 'n'
    elif nltk_tag.startswith('R'):
        return 'r'
    else:
        return 'n'

def remove_punctuation(text: str):
    return text.translate(str.maketrans('', '', string.punctuation))

def remove_numbers(text: str):
    return text.translate(str.maketrans('', '', string.digits))

def remove_urls(text: str):
    url_pattern = r'(http[s]?://\S+|www\.\S+)'
    return re.sub(url_pattern, '', text)

def remove_usernames(text: str):
    username_pattern = r'@\w+'
    return re.sub(username_pattern, '', text)

def convert_emoji_to_text(text: str):
    return emoji.replace_emoji(text, replace=' ')

def remove_whitespace(text: str):
    return re.sub(r'\s+', ' ', text).strip()

def remove_nonwords(text: str):
    return re.sub(r'[^\w\s]', '', text) #not sure, should remove hashtags?

def lemmatize(text: str):
    tokens = word_tokenize(text)
    # tokens = [token for token in tokens if token not in stopwords and token in valid_words]
    tokens = [token for token in tokens if token not in stopwords]
    pos_tagged = pos_tag(tokens)
    lemmatized_tokens = [lemmatizer.lemmatize(token, nltk_pos_tagger(tag)) for token, tag in pos_tagged]
    return lemmatized_tokens

def preprocess_data(text: str):
    text = remove_urls(text)
    text = remove_usernames(text)
    text = text.lower()
    text = remove_punctuation(text)
    text = remove_numbers(text)
    text = convert_emoji_to_text(text)
    text = remove_whitespace(text)
    tokens = lemmatize(text)
    return tokens

Pros and Cons of Lowercasing

Pros:

  1. Consistency: Ensuring that words like "Apple" and "apple" are treated as the same token, improving the model's ability to learn patterns.
  2. Vocabulary Size Reduction: Reduces the vocabulary size, which simplifies the model and reduces memory usage.
  3. Improved Generalization: Helps the model generalize better by treating different capitalizations of the same word as a single token.

Cons:

  1. Loss of Information: Capitalization can carry important information (proper nouns, acronyms).
  2. Ambiguity: Lowercasing can introduce ambiguity where words with different meanings are treated as the same ("US" vs "us").

Lowercasing is done to create uniformity in the text data, which helps in reducing complexity and improving the performance of the model by ensuring consistent handling of words. It is a trade-off between simplifying the text data and retaining potentially useful information.

Pros and Cons of removing numbers

Pros:

  1. Noise Reduction: Removes irrelevant numeric data that might not contribute to understanding the context or meaning.
  2. Simplified Vocabulary: Reduces the vocabulary size, which can make the model simpler and more efficient.
  3. Consistency: Ensures consistency in text representation, especially when numbers are not meaningful in the given context.

Cons:

  1. Loss of Information: Numeric data can be important for context, such as dates, quantities, or specific identifiers ("COVID-19", "Chapter 4").
  2. Reduced Context: In some cases, numbers provide essential context that can help in understanding the text better.
  3. Ambiguity: Removing numbers can make it harder to differentiate between texts where numbers are crucial ("Room 101" vs "Room 102").

Keeping hashtags

  1. Contextual Significance: Hashtags often encapsulate the main topic or sentiment of the tweet, which can be crucial for understanding its meaning.
  2. Keyword Indicators: Hashtags can act as important keywords that highlight specific issues, trends, or events related to the tweet's content.
  3. Relevance to Suicidal Prediction: In the context of predicting suicidal tendencies, hashtags may contain terms directly related to mental health, distress signals, or support mechanisms (#depression, #help).
  4. Social Media Context: Hashtags are a unique feature of social media language that carry meta-information about the tweet's content and should be preserved to maintain the integrity of the text.

Word2Vec - Word Embedding

Handling Out-of-vocabulary words

  1. Random Initialization
  • Pros: Simple to implement; provides a vector for every word.
  • Cons: May not capture any meaningful semantic information; adds randomness to the model.
  1. Use a Special Token (for instance "UNK")
  • Pros: Consistent handling of all unknown words; maintains model stability.
  • Cons: Loss of specific word information; treats all unknown words the same, reducing nuance.
  1. Character-level Embeddings
  • Pros: Can generate vectors for any word; captures subword information, useful for morphologically rich languages.
  • Cons: More complex to implement; may require more computational resources.
  1. Back-off to a Similar Word
  • Pros: Can provide a more meaningful vector than random initialization or a single "UNK" token.
  • Cons: Identifying a "similar" word might be computationally intensive; still not the exact representation.
  1. Train on a Larger Corpus or Update Vocabulary
  • Pros: Reduces the number of OOV words by expanding the dictionary.
  • Cons: Requires more data and computational resources; not always feasible.

class Twitter(Dataset):
    def __init__(self, dataframe: pd.DataFrame, w2v_model: gensim.models.KeyedVectors, sequence_len: int):
        self.dataframe = dataframe
        self.w2v_model = w2v_model
        self.max_sequence_len = sequence_len
        self.vector_size = w2v_model.vector_size
        self.df_token_col = "tokens"
        self._proc_dataset()
        self.len = len(self.dataframe)

    def __len__(self):
        return self.len

    def __getitem__(self, idx):
        return self.dataframe.iloc[idx]["vector"], self.dataframe.iloc[idx]["intention"]

    def get_vector_size(self):
        return self.vector_size

    def _proc_dataset(self):
        # Preprocess and return tokens list
        self.dataframe[self.df_token_col] = self.dataframe["tweet"].map(preprocess_data)

        # delete samples with empty tokens
        lwz = len(self.dataframe)
        self.dataframe = self.dataframe[self.dataframe[self.df_token_col].map(len) > 0]
        self.dataframe.reset_index(drop=True, inplace=True)
        print(f"Deleted 0-Len Samples: {lwz - len(self.dataframe)}")

        # Add padding
        self.dataframe[self.df_token_col] = self.dataframe[self.df_token_col].map(self._pad)

        # Get embedding's vectors
        self.dataframe["vector"] = self.dataframe[self.df_token_col].map(self._get_word_vectors)

    def _get_word_vectors(self, tokens: list) -> torch.tensor:
        # TODO: Return a 2D tensor for whole list of tokens, using vectors from w2v as explained on the description
        matrix = []
        for token in tokens:
            if token in self.w2v_model:
                matrix.append(self.w2v_model[token])
            else:
                matrix.append([0]*self.vector_size)
        return torch.tensor(matrix, dtype=torch.float)


    def _pad(self, tokens: list):
        # TODO: Add paddings (zero-vectors) into the end of sequence to reach the desired length
        if (len(tokens) >= self.max_sequence_len):
            tokens = tokens[:self.max_sequence_len]
        else:
            tokens = tokens + [""]*(self.max_sequence_len - len(tokens))
        return tokens

    def seq_report(self):
        length_all = self.dataframe[self.df_token_col].map(len).tolist()
        max_length = np.max(length_all)
        print(f"Sequence Length Report")
        print(f":::::MAX  LENGTH:::[{max_length:^5}]")
        print(f":::::MIN  LENGTH:::[{np.min(length_all):^5}]")
        print(f":::::MEAN LENGTH:::[{np.mean(length_all):^5}]")

        all_tokens = set()
        for token_set in self.dataframe[self.df_token_col].tolist():
            all_tokens = all_tokens.union(set(token_set))
        unique_tokens_count = len(all_tokens)
        valid_tokens = sum(1 if token in self.w2v_model else 0 for token in all_tokens)
        print("Sequence Tokenization Report")
        print(f":::::All Unique Tokens:::[{unique_tokens_count:^6}")
        print(f":::::All Valid Tokens:::[{valid_tokens:^6}")
        print(f":::::Valid Tokens:::[{round(100*valid_tokens/unique_tokens_count, 2):^5}%]")

    @staticmethod
    def _to_tensor(tokens: list):
        return torch.tensor(tokens, dtype=torch.float32)

def model_eval(model, loader, loss_function, device: str = ['cuda', 'cpu', 'auto']):
    """Returns test_loss, test_acc"""

        device = torch.device("cuda" if torch.cuda.is_available() else "cpu")

    model = model.to(device)
    itr = tqdm(loader, total=len(loader), leave=True)

    for inputs, labels in itr:
        # TODO: move model's inputs to `device`
        inputs, labels = inputs.to(device), labels.to(device)

        # TODO: use model's forward pass to generate outputs
        outputs = model(inputs)

        # TODO: calculate model's loss
        loss = loss_function(outputs, labels)
        test_loss += loss.item()

        # TODO: calculate/update model's accuracy
        _, predicted = torch.max(outputs, 1)
        correct_samples += (predicted == labels).sum().item()
        total_samples += labels.size(0)
        test_acc = correct_samples / total_samples

        itr.set_description("(Eval)")
        itr.set_postfix(
            loss=round(loss.item(), 5),
            accuracy=round(test_acc, 5),
        )
    test_loss /= len(itr)
    return test_loss, test_acc

import torch.nn.functional as F
import torch
import torch.nn as nn

class CNN(nn.Module):
    def __init__(self, context_window=64, regularization=False):
        super(CNN, self).__init__()
        self.regularization = regularization
        self.context_window = context_window

        self.conv1 = nn.Conv1d(in_channels=self.context_window, out_channels=64, kernel_size=3, padding=1)
        self.conv2 = nn.Conv1d(in_channels=self.context_window, out_channels=64, kernel_size=5, padding=2)
        self.conv3 = nn.Conv1d(in_channels=self.context_window, out_channels=64, kernel_size=7, padding=3)

        self.dropout = nn.Dropout(0.5)

        self.conv4 = nn.Conv1d(in_channels=64, out_channels=128, kernel_size=3, padding=1)
        self.conv5 = nn.Conv1d(in_channels=64, out_channels=128, kernel_size=5, padding=2)
        self.conv6 = nn.Conv1d(in_channels=64, out_channels=128, kernel_size=7, padding=3)

        self.bn4 = nn.BatchNorm1d(128)
        self.bn5 = nn.BatchNorm1d(128)
        self.bn6 = nn.BatchNorm1d(128)

        self.maxpool = nn.MaxPool1d(kernel_size=2)
        self.global_avg_pool = nn.AdaptiveAvgPool1d(1)

        self.flatten = nn.Flatten()

        conv_output_size = 128 * 3
        self.fc1 = nn.Linear(in_features=conv_output_size, out_features=128)
        self.fc2 = nn.Linear(in_features=128, out_features=2)

    def forward(self, x):

        x1 = F.relu(self.conv1(x))
        x2 = F.relu(self.conv2(x))
        x3 = F.relu(self.conv3(x))

        if (self.regularization):
          x1 = self.dropout(x1)
          x2 = self.dropout(x2)
          x3 = self.dropout(x3)

        x4 = None
        x5 = None
        x6 = None

        if (self.regularization):
          x4 = F.relu(self.bn4(self.conv4(x1)))
          x5 = F.relu(self.bn5(self.conv5(x2)))
          x6 = F.relu(self.bn6(self.conv6(x3)))

        else:
          x4 = F.relu(self.conv4(x1))
          x5 = F.relu(self.conv5(x2))
          x6 = F.relu(self.conv6(x3))

        if (self.regularization):
          x4 = self.dropout(x4)
          x5 = self.dropout(x5)
          x6 = self.dropout(x6)

        x4 = self.maxpool(x4)
        x5 = self.maxpool(x5)
        x6 = self.maxpool(x6)

        x4 = self.global_avg_pool(x4)
        x5 = self.global_avg_pool(x5)
        x6 = self.global_avg_pool(x6)

        x = torch.cat((x4, x5, x6), dim=1)

        x = self.flatten(x)

        x = F.relu(self.fc1(x))
        x = self.fc2(x)

        return x

Adam Optimizer

Adaptive Moment Estimation Combines the benefits of two other extensions of stochastic gradient descent (SGD). Specifically, Adam computes adaptive learning rates for each parameter.

  • Key Features:
    • Adaptive Learning Rate: Adjusts learning rate individually for each parameter.
    • Momentum: Uses running averages of both the gradients and their squares (first and second moments).

Difference with SGD:

SGD:

  • Learning Rate: Fixed or decayed manually over time.
  • Momentum: Can be added, but it's not inherent.
  • Convergence: Slower and more sensitive to initial learning rate.

Adam:

  • Learning Rate: Adaptive, reducing the need for manual tuning.
  • Momentum: Inherent, uses first and second moments of gradients.
  • Convergence: Faster and more robust to noisy data and sparse gradients.

Cross Entropy Loss for Classification Problems

  1. Suitability: Cross Entropy Loss is well suited for multi class classification tasks. It penalizes predictions that are confidently wrong more heavily.
  2. Probability Interpretation: It compares the predicted probabilities of each class with the true distribution of classes. It provides a measure of how well the model's predicted probabilities match the true distribution.
  3. Differentiability: It's differentiable, making it suitable for gradient-based optimization algorithms like backpropagation.
  4. Overall Performance: It has been shown to work well in various classification tasks, including suicidal prediction, by effectively penalizing incorrect predictions and guiding the model towards better performance.

Train Test partitioning

A 20% test partition is commonly used and generally provides a reasonable balance between training and testing data. It provides enough instances for evaluating model performance without sacrificing too much training data. It allows for robust evaluation of the model's generalization performance on unseen data. it helps mitigate the risk of overfitting by providing a larger portion of data for training.

Kernel size

  1. Feature Extraction:
    • Low Kernel Size: Captures fine-grained details and local patterns.
    • High Kernel Size: Encodes broader spatial information and global patterns.
  2. Local vs. Global Information:
    • Low kernel sizes focus on local neighborhoods, extracting fine details.
    • High kernel sizes consider larger regions, capturing more global features.
  3. Model Complexity:
    • Low kernel sizes result in fewer parameters and less computational complexity.
    • High kernel sizes increase model capacity and computational requirements.
  4. Interpretation:
    • A low kernel size means the model concentrates on intricate details, suitable for tasks with small-scale features.
    • A high kernel size implies a broader scope, beneficial for detecting larger patterns or structures in the data.

Output Reduction in Convolutional Layers

  • Pros:

    • Dimensionality Reduction: Applying pooling operations like max pooling or global average pooling after convolutional layers reduces the spatial dimensions, focusing on the most informative features.
    • Non-linearity: Convolutional layers followed by activation functions introduce non-linearities, enhancing the model's capacity to capture complex patterns.
    • Hierarchical Feature Extraction: Convolutional layers progressively extract higher-level features, enabling the model to learn hierarchical representations.

  • Advantages Over Feedforward Layers:

    • Spatial Hierarchical Representation: Convolutional layers capture spatial relationships in the input data, facilitating the extraction of local patterns.
    • Parameter Efficiency: Fewer parameters are required compared to fully connected layers, reducing the risk of overfitting and computational complexity.
    • Translation Invariance: Convolutional layers exploit local receptive fields, making the model more robust to translations in the input data.

  • Choice Justification:

    • Performance: Convolutional layers are effective in capturing spatial dependencies and reducing dimensionality, contributing to improved performance and generalization.
    • Feature Abstraction: By abstracting features hierarchically, convolutional layers enable the model to learn more discriminative representations, enhancing its predictive capability.

Setting Context Window Larger than Input Tweets

Pros:

  • Uniformity: Ensures all inputs are processed with a consistent context window, simplifying model architecture.
  • No Truncation: Prevents loss of information from longer texts, as the context window will always be sufficient.

Cons:

  • Wasted Resources: Excessive padding for shorter texts can lead to inefficient use of computational resources and memory.
  • Noise Introduction: Too much padding can introduce irrelevant information, potentially degrading model performance.
  • Overfitting Risk: Larger context windows may increase the number of parameters, raising the risk of overfitting, especially with limited data.
